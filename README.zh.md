## 用"意思"搜你的文件

你记得**某个文件是讲什么的**。但你记不住它的名字、在五个盘的哪一个、当初写的是"发票"还是"invoice"还是"账单"。关键词搜索恰恰在这里失灵。

**Scry 按意思搜索。** 它的核心是一个**专为语义打造的多语模型**——一个仅 **1.18 亿参数的嵌入网络**，跨 **100+ 种语言**理解*文字到底在说什么*，**而不是**一个几个 GB 的本地大语言模型硬塞进搜索框。它用这个模型对**全盘候选进行毫秒级神经网络重排**；当你要最好的质量时，它还能对你选定的文件夹**建立完整的深度语义索引**——把每一段文字都嵌入向量,于是你搜的"**意思**"能命中那些和你查询词**一个字都不重合**的文件——**甚至跨语言**（用中文搜，也能搜到英文文件，反之亦然）。

一个**真正理解语义的引擎**,不是贴了 AI 标签的关键词匹配——它的模型**只有 235 MB、轻量、100% 跑在你自己的机器上。**

> 这就是 **Everything** 本该进化成的样子：一样的"全盘、即时"手感——但它听得懂你。

## 一个搜索框，三种搜法

| 模式 | 干什么 | 需要设置 |
|---|---|---|
| 🌐 **全盘语义** | 按意思搜全机器每个文件的**内容**。骑 Windows 内容索引拿候选，再用神经网络按意思重排。 | **不用。** 打开即搜。 |
| ⚡ **文件名** | 像你熟悉的那样，全盘秒搜文件名。 | 不用。 |
| 🎯 **深度语义（掌控模式）** | 指定文件夹（或整个盘），它建**自己的**深度索引——质量最高，能命中**零字面重合**的改写。首次全量扫描，之后只碰有变化的文件，并在后台自动保持最新。 | 每个文件夹点一下。 |

> **第一次用？先选对模式。**「全盘语义」即时、免设置，但它的候选来自 Windows 索引——所以一个**纯改写或跨语言、零字面重合**的查询（正是最能"哇"的那种 demo）在这里可能一条都搜不到。这种就该用「深度语义」：把文件夹掌控一次、建好索引，它就能在**一个字都不重合**时照样命中意思。一句话——**全盘语义 = 广而即时；深度语义 = 意思最深。**

此外：**中英双语界面（一键切换）**、全键盘操作（`回车`搜索、`↑/↓`移动、`回车`打开）、高亮片段直接告诉你**为什么**这个文件匹配，还有一个**免装 Python** 的便携版，在任意 Windows 电脑上双击即用。

**可选扩展包（DLC）。** 基础版保持小巧、100% 离线;重的增强（首个是更大的多语模型）**只在你点击时**才从本仓库下载并校验 SHA‑256——在界面「扩展包」里。

**连图片里的文字也能搜（可选 OCR）。** 在掌控模式里打开 OCR，Scry 就用 **Windows 自带的 OCR 引擎**读出你的**截图、扫描件、照片**里的文字——一张收据截图、一份扫描合同,都能按**意思**被搜到,语言取决于你 Windows 装了哪些 OCR 语言包(大多数机器开箱就有中文 + 英文)。它**不下载任何模型**(引擎和语言数据都由 Windows 提供)、**全程离线**、只给程序加约 4 MB,而且**默认关闭**——打开后点「重新扫描」即可。它的定位是"**找得到**",不是完美转录:花哨或低清的文字可能识别得有点糙,但语义搜索并不在意。

## 它会越用越懂你 —— 在你自己的机器上

你越用，它越贴合你：**你经常打开的文件和文件夹，会在相似的搜索里自动往前排**，并偏向你最近的习惯。这只是温和的重排——**绝不劫持一次全新的查询，也绝不把更相关的结果压下去**。而且这是**零代价**的个性化：它只从**你自己的点击**学习、只存在**你自己的磁盘**上，**从不训练模型、也从不把任何东西传出去**。随时可以关掉，或点「忘记我」一键清空。

## 隐私立场（这就是全部意义）

- **只读。** Scry 只会*读*你的文件来建索引——**从不修改、移动、重命名或删除**它们。你的原文件分毫不动。
- **零外联。** 基础版根本不联网：只监听 `127.0.0.1`（连局域网都不开放），模型已内置，也没有服务器可联——无账号、无遥测、无"匿名使用统计"。拔掉网线一切照常。Scry **唯一**一次碰网络，是*你*主动点下载可选扩展包时——从本仓库的 GitHub Releases 下载并校验 SHA‑256。（图片 OCR 用的是本机 Windows API，也不联网。）
- **全程本地存储。** 你的索引和设置存在自己磁盘的 `~/.localsearch_index`；删掉那个文件夹就清空一切。
- **占用很轻。** 纯 CPU（无需显卡）；内置模型约 235 MB，10 万文件的深度索引约 154 MB。全盘语义 / 文件名两种模式完全不建索引。

## 工作原理

```
        你的查询（"那个水库跌到历史最低的文件"）
                              │
        ┌─────────────────────┼───────────────────────────┐
     全盘                   文件名                      深度语义
   (Windows 内容索引)    (Everything 式，MFT 文件名)   (对你选的文件夹建的 FAISS 索引)
        │                                                    │
        └──────────────►  神经网络重排 · Scry 语义模型  ◄──────────┘
                              │
                     排好序的文件 + 精确到命中的那一句，高亮显示
```

- **Scry 语义模型：** 一个仅 **1.18 亿参数的多语嵌入网络**（`multilingual-e5-small`，384 维，**100+ 种语言**），专门用来表示*意思*——不是生成文本的大模型。纯 CPU 可跑、无需显卡，且**从不拿你的文件去训练**。
- **检索：** 精确 FAISS（fp16 标量量化——内存/磁盘减半，仍是穷举精确）+ BM25 词法一臂融合；CJK 感知分块 + 二元组分词，中文文档能被完整搜到。
- **即时模式：** 通过 `Search.CollatorDSO` 直查系统内容索引，再嵌入重排——**零预索引**的全盘内容搜索。
- **稳如老狗：** 索引写入防掉电（fsync + 代号戳，撕裂的保存能被检测并隔离）、退出前自动保存、建索引期间进度条实时刷新且能同时搜索、单线程看护器扛得住批量拷贝。

Scry 是**多语**的——一个模型覆盖 100+ 种语言，所以你能用一种语言搜、找到另一种语言写的文件（中文和英文都是一等公民）。跨语言检索很强但不是魔法：偶尔会输给一篇话题相近的同语言文件。它的目标不是在跑分上击败数据中心，而是给你**云端级的便利 + 零隐私代价**。

## 项目结构

```
localsearch/          引擎 + 本地网页界面（这就是产品本体）
  ├─ engine.py        总调度：扫描 → 解析 → 分块 → 嵌入 → 索引 → 搜索
  ├─ index.py         双索引：fp16 FAISS 稠密 + BM25，防掉电持久化
  ├─ winsearch.py     "即时全盘"的关键突破（Windows 内容索引）
  ├─ search.py        混合检索 + 最佳句片段
  ├─ watcher.py       防抖后台增量重索引
  ├─ personalize.py   本地、免训练的重排（从你打开过的文件学习）
  ├─ app.py           Flask 后端（仅 127.0.0.1）+ JSON API
  └─ web/index.html   单页界面，双语，零构建步骤
demo_*.py             可直接运行的端到端示例（含 demo_personalize.py）
run.bat               双击启动
```
